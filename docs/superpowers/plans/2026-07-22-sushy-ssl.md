# Sushy SSL/TLS Support Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add HTTPS on port 8443 alongside existing HTTP on port 8000 for sushy-emulator (Redfish BMC), enabling RHSE and other consumers that require TLS on the Redfish endpoint.

**Architecture:** Per-VM self-signed certs generated at BMC setup time via `openssl` CLI. Both HTTP (8000) and HTTPS (8443) always run in parallel — no toggle, purely additive. Existing teardown/restore/status handlers already use glob patterns (`sushy-*.pid`, `sushy-*.conf`) that pick up the new SSL files automatically.

**Tech Stack:** Python stdlib (`subprocess`, `ssl`), openssl CLI, sushy-tools (`SUSHY_EMULATOR_SSL_CERT`/`SUSHY_EMULATOR_SSL_KEY` config), Python `http.server` with `ssl.SSLContext` (KubeVirt)

## Global Constraints

- troshkad is stdlib-only — no pip imports, use `subprocess.run(["openssl", ...])` for cert generation
- KubeVirt entrypoint runs on UBI9 python-311 — `openssl` CLI is available
- Self-signed certs with 10-year validity (ephemeral lab environments)
- EC P-256 keys (same as existing troshkad TLS certs in `agent_deployer.py`)
- SAN must include the BMC IP address for clients that check it
- Port 8443 for HTTPS (8000 stays HTTP)

def _generate_self_signed_cert(cert_path, key_path, cn, ip):
    """Generate a self-signed TLS certificate with the given CN and IP SAN."""
    subprocess.run(
        [
            "openssl", "req", "-x509",
            "-newkey", "ec", "-pkeyopt", "ec_paramgen_curve:prime256v1",
            "-nodes", "-days", "3650",
            "-subj", f"/CN={cn}",
            "-addext", f"subjectAltName=IP:{ip}",
            "-keyout", key_path,
            "-out", cert_path,
        ],
        capture_output=True,
        timeout=10,
        check=True,
    )
    os.chmod(key_path, 0o600)
```

def _generate_self_signed_cert(cert_path, key_path):
    """Generate a self-signed TLS certificate."""
    import subprocess
    subprocess.run(
        [
            "openssl", "req", "-x509",
            "-newkey", "ec", "-pkeyopt", "ec_paramgen_curve:prime256v1",
            "-nodes", "-days", "3650",
            "-subj", "/CN=troshka-bmc",
            "-keyout", key_path,
            "-out", cert_path,
        ],
        capture_output=True,
        timeout=10,
        check=True,
    )


if __name__ == "__main__":
    import ssl
    import threading

    # HTTP server on existing port (default 8000)
    http_server = HTTPServer(("0.0.0.0", LISTEN_PORT), RedfishHandler)

    # HTTPS server on port 8443
    ssl_port = int(os.environ.get("SUSHY_SSL_PORT", "8443"))
    cert_path = "/tmp/sushy.crt"
    key_path = "/tmp/sushy.key"
    _generate_self_signed_cert(cert_path, key_path)

    https_server = HTTPServer(("0.0.0.0", ssl_port), RedfishHandler)
    ctx = ssl.SSLContext(ssl.PROTOCOL_TLS_SERVER)
    ctx.load_cert_chain(cert_path, key_path)
    https_server.socket = ctx.wrap_socket(https_server.socket, server_side=True)

    ssl_thread = threading.Thread(target=https_server.serve_forever, daemon=True)
    ssl_thread.start()
    print(f"Redfish emulator listening on port {LISTEN_PORT} (HTTP) and {ssl_port} (HTTPS)")
    http_server.serve_forever()
```
